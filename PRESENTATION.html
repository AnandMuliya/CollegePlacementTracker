PROJECT OVERVIEW

Purpose

Streamline and automate college placement activities for students, HODs, and Training & Placement Officers through a centralized digital platform.

Target Users

  • Students - Apply & Track
  • HODs - Monitor & Approve
  • TPO - Manage & Oversee

Key Objectives

  • Centralized placement data management system
  • Role-based access control with security
  • Real-time application tracking and notifications
  • Enhanced security with data encryption
  • Modern user experience with Material Design
2 / 8

SYSTEM ARCHITECTURE

MVVM Pattern with Repository Layer

Presentation Layer

Activities, ViewModels, LiveData for reactive UI updates

Domain Layer

Repository classes, business logic, use cases

Data Layer

Room Database, DAOs, Entities (User, Company, Application)

Utility Layer

  • SecurityUtils - PBKDF2 encryption
  • ValidationUtils - Input validation
  • UIHelper - User feedback system
  • DateUtils - Date operations
  • NotificationHelper - Push notifications
3 / 8

DATABASE SCHEMA

Table Key Fields Purpose
User id, email, password, role, branch, cgpa Authentication and profile management
Company id, companyName, packageAmount, minimumCGPA Company details and eligibility criteria
Application id, studentId, companyId, status Application tracking and status management
Student id, rollNumber, skills, projects Extended student information

Database Features

  • Foreign key constraints for data integrity
  • Indexed fields for optimized queries
  • LiveData for reactive updates
  • Automatic timestamps and versioning
